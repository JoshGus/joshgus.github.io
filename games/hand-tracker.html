Implementation notes

MediaPipe Hands
Uses Google's @mediapipe/hands model — a TensorFlow Lite graph running entirely in-browser via WebAssembly. No server. The model returns 21 3D landmarks per hand (wrist + 4 joints per finger) at up to 30 fps on typical hardware.
Pipeline
The webcam feed runs into a <video> element. Each frame, the MediaPipe Hands solution processes the video and calls a result callback with detected landmarks. The callback drives both the canvas overlay renderer and the side panel data display.
Canvas overlay
A transparent <canvas> is absolutely positioned over the video. The renderer draws landmark dots and connecting bone lines by mapping the normalised 0–1 coordinates to canvas pixels. The video is CSS-mirrored (scaleX(-1)), so the canvas draw calls mirror too.
Coordinate system
MediaPipe returns x, y normalised to [0, 1] of the video frame, plus a relative z depth (negative = closer). The panel displays x/y directly. Depth is not currently used — it's available as a foundation for gesture-controlled interactions.
Fingertip tracking
The five fingertip landmarks (indices 4, 8, 12, 16, 20 in the 21-point hand skeleton) are extracted and displayed in the side panel with live coordinate updates. These are the natural attachment points for pointer-based gesture input in future experiments.
Design intent
This page is a foundation layer — a verified working pipeline from webcam → model → landmarks → display. The next step is attaching game input to landmark positions, replacing mouse/touch with hand gestures for the game pages on this site.
