Implementation notes

Three steering rules
Separation: push away from any neighbor closer than sepRadius. Alignment: nudge velocity toward the average velocity of neighbors within visRadius. Cohesion: steer toward the average position of visible neighbors. Each rule produces an acceleration vector; they're summed and applied to velocity each frame.
Complexity
Neighbor search is O(n²) per frame — every boid checks every other. For n = 500 that's 250,000 distance comparisons per frame. Squared distances are computed first; Math.sqrt is only called when needed for the separation direction vector. A spatial hash grid would reduce this to O(n) but adds significant code for a problem that fits comfortably in O(n²) at this scale.
3D projection
Boids live in a fixed world volume (800 × 600 × 800 units). A Y-axis rotation matrix rotates the world each frame: rx = x·cos(θ) − z·sin(θ). Perspective divide: scale = fov / (depth + fov) where depth is the rotated z. Screen position scales with canvas.width / worldWidth so the flock fills the viewport regardless of window size.
Speed clamping
After summing all steering accelerations, velocity is clamped to [minSpeed, maxSpeed] by rescaling the vector: v = v / |v| · clampedSpeed. Without a minimum speed, boids can stall when forces cancel out. Without a maximum, separation impulses on close encounters cause boids to fly off-screen.
Clear strategy
The canvas is hard-cleared each frame with a solid background fill. A motion trail (semi-transparent fill) was tried but creates artifacts during camera orbit — the smear is in screen space, so rotating the view leaves ghost streaks that don't correspond to any spatial position. Clean clear + bounding box gives better spatial orientation instead.
Depth sort
Boids are sorted back-to-front by their rotated z coordinate before drawing, so closer boids correctly overlap farther ones. A new sorted array is created each frame via [...boids].sort() — O(n log n). The original array is left in insertion order for the O(n²) physics update, which doesn't require any particular ordering.
Boundary condition
Soft wall avoidance: within a 90-unit margin of each face, a repulsive force proportional to penetration depth steers boids back toward the center. Force = tf × (overlap / margin). A hard clamp zeroes the inward velocity component if a boid somehow reaches the wall face — this is the safety net, not the primary mechanism. Produces natural flocking near walls without the flicker of hard reflection.
Dynamic count
The count slider adds or removes boids immediately. Adding creates a new Boid at a random position with a velocity in the current speed range. Removing truncates the array from the end — O(1). No object pooling needed at this scale; GC pressure from creating 300 new objects on a slider drag is negligible.
