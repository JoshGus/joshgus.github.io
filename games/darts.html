Implementation notes

Hand detection
Uses Google's @mediapipe/tasks-vision HandLandmarker running fully in-browser via WebAssembly. Returns 21 3D landmarks per hand at up to 30 fps with no server.
Aiming
Index fingertip (landmark 8) maps to the dartboard. Camera feed and landmarks are mirrored so movement feels natural. Coordinates are smoothed with exponential averaging (factor 0.2) to prevent raw jitter from showing on the crosshair.
Throw detection
Palm scale — 2D distance from wrist (landmark 0) to middle knuckle (landmark 9) — is tracked per frame. A >22% increase over 7 frames indicates the hand is moving rapidly toward the camera, triggering a throw. 1.2s cooldown prevents doubles.
Accuracy scatter
Aim position variance over the last 12 frames is measured before each throw. The landing offset is a random vector scaled by that variance. Steadier aiming = tighter grouping. Click the board or press Space as an alternative.
Depth as stability
The aim circle radius reflects current jitter magnitude. Closer hands tend to produce more stable landmark readings (larger palm in frame → more pixels per landmark), so the depth-to-accuracy correlation is an emergent property of the model, not an explicit calculation.
2-player ready
Before a game, each player holds up fingers: 1 finger confirms P1, 2 fingers confirms P2. Requires a 1-second hold to prevent accidental triggers. Both players can stand in frame simultaneously.
301 scoring
Each player starts at 301. Darts subtract from the running total. Going below zero is a Bust — the round score is forfeited and the score resets to what it was at the start of the turn. First player to reach exactly 0 wins.
Webcam overlay
The raw webcam feed is layered behind the game canvas with CSS scaleX(-1) mirroring. The opacity slider fades it in to help players check their hand position relative to the aiming cursor.
