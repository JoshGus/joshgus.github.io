Brush 3

Implementation notes

Grid storage
Flat Uint8Array(cols × rows) — one byte per cell, row-major layout. A 2D array would be cleaner API but the flat array gives better cache locality during the bottom-to-top sweep and halves the pointer-chasing overhead.
Simulation order
Cells are processed bottom-to-top so a falling grain of sand moves into already-processed space below it — no double-move per frame. Horizontal direction randomizes each frame (left-to-right or right-to-left) to prevent directional bias in spreading materials.
Double-step prevention
A second Uint8Array called updated tracks which cells moved this frame. Fire rising into an unprocessed cell above gets marked updated immediately, so the sweep won't step it a second time when it reaches that row. O(cols × rows) reset per frame via TypedArray.fill(0).
Fire lifetime
Fire uses the meta byte array to store per-cell lifetime (0–180). Each step decrements it; at zero the cell becomes air. Color transitions through white-yellow → orange → red as lifetime falls. Fire spreading to wood uses Math.random() < 0.008 per neighbor per frame — slow enough to look natural.
Rendering
ctx.createImageData pre-allocates a Uint8ClampedArray the size of the canvas. Each frame, the entire buffer is written and pushed via putImageData — one GPU upload per frame rather than thousands of fillRect calls. CELL = 4px per cell; the buffer write is O(pixels) = O(cols × rows × 16).
Color variation
Material colors use a position hash (c·7 + r·13) & 31 for deterministic per-cell texture variation — no RNG call in the hot render loop. Fire flickers by including a global frame counter in the hash, updating color every frame.
Water & lava flow
Both try to fall first, then spread horizontally up to 4 cells. Lava spreads with a 30% chance per frame (viscosity). Lava touching water solidifies both into stone. The horizontal spread iterates outward until it hits a non-empty cell — effective O(spread_width) per cell per frame.
Inspired by
Powder Game (dan-ball.jp) and Noita — Noita extends this concept to a full game where every pixel is physically simulated. This implementation follows the same cellular-automata approach at a much smaller scale.
